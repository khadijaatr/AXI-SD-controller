# AXI-SD-controller
Abstract

This project focuses on the design and verification of an SD card controller for an external memory card, intended as part of a SoC environment for neural signal processing. 
The main purpose of the controller is to enable the system to store and retrieve data through an external SD card, allowing recorded information to be preserved for later analysis.
The broader target system is designed to support research in neural-level treatments by recording brain activity from laboratory mice under different stimulation conditions. In such a system, sampled neural data may be collected by dedicated acquisition blocks, stored temporarily in internal memory, and later transferred to an external SD card for documentation, analysis, and post-processing.
In the implemented project, the controller acts as a bridge between the system side and the SD card side. The system communicates with the controller through an AXI-based register interface, while the SD card side is connected through an SDIO interface using command and data lines. 
The controller receives read and write requests from the AXI side, translates them into SD card operations, and performs block-based data transfers using the supported SD commands.
The current implementation supports the essential block transfer operations required for the project demonstration: card reset, single-block write, and single-block read. These operations are implemented using CMD0, CMD24, and CMD17 respectively. An internal block buffer is used to hold data during transfers between the AXI interface and the SDIO interface.
The project was verified using a SystemVerilog simulation environment. The testbench connects an AXI master bus functional model, the SD-card controller, an SD bus interface, and an SD card bus functional model. The verification flow writes data from the AXI side into the SD card model, reads it back through the controller, and compares the returned data with the original data. The simulation passed successfully, demonstrating correct end-to-end data transfer between the system interface and the SD card model.
The implemented design provides a simple, clear, and extendable controller architecture. While it does not implement a complete commercial SD card controller, such as full initialization, CRC generation/checking, FIFO streaming, or full 512-byte sector handling, it establishes the required foundation for system-to-SD communication and can be extended in future work toward a more complete and hardware-ready SD controller.
